Unity-00入门
2021-02-01
Word count: 1.5k|Reading time: 5 min

概述

前些天完成作业临场突击 Unity，这里简要总结一下

总功能：控制人物移动，进行攻击

完成功能需要涉及到：

  • 素材的导入
  • 素材“封装”成prefabs
  • 给物体增加物理特性：刚体、碰撞
  • 脚本与物体的结合

功能

首先，导入素材不多讲，直接拖到页面中即可

对素材“赋予”性质、功能（碰撞、刚体）

直接导入的素材，比如 png, jpg 等只是单纯的图片，并不具备我们需要的性质。比如我们导入了一张地面的图片，但是这个“地面”并不能将人承载起来，导入了一个球的图片，“球”也不具备重力。

Q：如何给素材赋予这些能力？
A：选中素材(以二维举例)，然后菜单栏：Component->Physics 2D->Rigidbody 2D

这样，该素材就具备了“碰撞”的能力

同样，对于一个刚体（重力等）可以添加 Rigidbody 2D 组件

在以上例子中，想要实现人站在地上的功能，需要：

  • 人: Rigidbody 2D, Rigidbody 2D
  • 地: Rigidbody 2D

其余的功能或大小性质等类似

预制体

以上的步骤已经可以满足我们的需要，但如果同样一类物品需要大量拖入场景使用，如果每一个都像以上所述单独添加组件等进行调整，无疑十分繁琐。

所以我们可以对素材进行封装：

  1. 将素材拖入场景，对其进行组件的添加以及其他性质的调整
  2. 在 Scene 中选中物体，可以发现对应 Hierarchy 窗口中也被选中
  3. 将 Hierarchy 中被选中的条例拖入 Project 中：可以在 Assets 中新建一个文件夹 Prefabs 等，然后这样被封装后的产物就可重复使用

不同预制体间的层次问题可以在 工具栏->Edit->Project Settings->Physics 2D 下方的 Layer Collision Matrix 中设置

C#脚本

想要实现方向键控制人物移动以及攻击等功能，都需要编写脚本来实现

目前掌握的都是：一类物体对应一个脚本。比如：

  • 玩家对应一个玩家的控制脚本
  • 敌人对应一个敌人的控制脚本
  • 子弹对应一个子弹的控制脚本

值得注意的是：

  • 脚本的名称和脚本内容的类名一定要匹配
  • 想要在一个脚本中引用其他的类，比如想要实现敌人撞到墙后改变运动方向，就必须要对墙也编写一个脚本，哪怕脚本为空，否则“墙”的存在无法在敌人脚本中得到体现
  • 编写完脚本后将脚本拖到对应物体的 Inspector 栏中实现“绑定”的效果

内容

脚本内容可以有很多，但目前实现的只有 4 个：

  1. Awake()： 早于 Start()
  2. Start()
  3. Update(): 角色方向的控制等都会在这里实现
  4. OnCollisionEnter2D(Collision other): 实现碰撞后行为，如子弹打中敌人后子弹消失，敌人血条下降等

行为

主要实现了：

  1. 方向键控制人物移动
  2. 控制产生另外的物体：人物发射子弹
  3. 不同物体的碰撞：敌人撞墙、玩家遇到敌人、子弹打中敌人

方向键控制人物移动

以一个方向为例：

  1. 监听键盘输入：自带对 方向键、wasd 的检测
  2. 获得方向向量：创建 Vector2 (以二维为例)
  3. 获得当前玩家位置: transform.position : transform 这个量很特殊，自带当前物体的一些性质（当前位置等）
  4. 根据键盘输入计算位移
  5. 设置位置

按下空格控制人物跳跃：

  1. 监听键盘输入
  2. 设置刚体的力、速度

控制产生另外的物体：人物发射子弹

  1. 监听键盘输入
  2. 根据最开始声明的 GameObject 量进行实例化产生物体：具体此 GameObject 是指产生什么物体（比如子弹）需要返回界面在对应玩家的 Inspector 中进行选择：可以直接将子弹的 预制体 拖入到相应选项中
  3. 获得产生物体的 类：上一步的产生的物体只是一个单纯的素材或是预制体，并不具备其他的性质。想要获得比如对子弹所设置的脚本中的其他方法还需根据产生的物体获取对应的类的对象
  4. 根据类进行其他的操作

不同物体的碰撞：敌人撞墙、玩家遇到敌人、子弹打中敌人

以敌人为例：

  1. 敌人有10滴血，每收到一次子弹打击扣一滴，当血扣完时死亡（消失）
  2. 敌人撞到墙后改变运动方向

如何获取碰撞自己的物体：

  1. OnCollisionEnter2D 参数中的 Collision2D 对象就是碰撞自己的物体
  2. 但是具体该碰撞物体的信息需要专门获取：根据 Collision2D 获取组件： other.gameObject.GetComponent()
  3. 然后就可根据获取的组件进行相应的判断和操作了
