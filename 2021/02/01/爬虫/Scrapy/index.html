Scrapy 入门

模块

  1. spiders
  2. items.py: 定义爬取数据的容器
  3. pipelines.py: Item 生成后会自动送到 pipeline 进行处理
  4. settings.py: 设置关联
  5. middlewares.py:

基本用法

  1. 首先通过命令行创建框架
  2. 利用命令行创建爬虫 spider， 一只 spider 负责爬一类资源，然后在创建的 py 中完善类的定义
    1. 属性&方法
      1. name:spider 名称
      2. allowed_domains:允许爬取的域名
      3. start_urls:启动时爬取的 url 列表
      4. parse(self, response):对获得请求 response 的解析方法
    2. parse 任务：
      1. 对爬取到的整个response进行解析，解析出每一条单独数据，然后 yield
      2. 最后得到 spider 的下一跳的 url，创建 scrapy.Request(url,callback),yield
        1. url:下一条的地址
        2. callback:对下一跳的解析方法，一般来说因为数据结构基本相同，所以采用 self.parse
  3. 在 items.py 中完善所爬取数据的容器（包装）：一个网页中爬取到的每一条数据的格式
  4. 通过命令行运行爬虫，即可看到爬虫的运行结果

基本命令行操作

  1. 创建 Scrapy 项目: scrapy startproject demo
  2. 创建 Spider: scrapy genspider spider1 example.spider.com
  3. 进行爬取、运行 spider: scrapy crawl spider1
  4. 将爬取结果保存到文件：
    1. json: test.json
    2. csv: test.csv
    3. xml: test.xml
    4. pickle: test.pickle
    5. marshal: test.marshal
    6. ftp: ftp://…/path/to/test.csv

Selector 的用法

Scrapy 提供了自己的数据提取方法

步骤也很简单：

  1. 直接利用 Selector 类构造一个选择器对象
  2. 调用相关方法提取数据，如：xpath(), css()

几个主要提取方法都可以使用：

  1. xpath
  2. css
  3. re

不过 re 与 xpath, css 选择的结果不同在于：

  1. xpath(), css() 提取的结果需要 extract()
  2. re 提取的结果直接就是列表
  3. re() 也有对应的 re_first()

Scrapy shell: 可以直接进入 scrapy shell, 交互界面模拟请求

  • 命令行中输入： scrapy shell http://…

值得注意的是 response.xpath(), response.css() 功能完全等同于 response.selector.xpath(), response.selector.css()

得到结果后，使用 extract() 后得到的都是列表，所以提取元素需要加下标

  • 使用 extract_first() 可以直接提取匹配的第一个结果（可设置默认结果）

xpath, css 可嵌套选择

Spider 的用法

可以发现，在 Scrapy 中，最核心的类就是 Spider 了。要抓取网站的链接配置、抓取逻辑、解析逻辑都是在 Spider 中配置的。

整个爬取的循环过程：

  1. 用初始 URL 初始化 Request, 成功请求返回时，Response 生成并作为参数传给 callback
  2. callback 内分析网页内容，返回
    1. Item
    2. 链接构造下一个 Request
  3. 如果返回的是字典或 Item 对象，可通过 Feed Exports 等组件存入文件；如果设置了 Pipeline, 可使用 Pipeline 处理并保存
  4. 如果返回的是 Request, 那么执行成功后得到 Response 后，Response 会传递给 Request 中定义的回调函数，在回调函数中我们可以再次使用选择器分析新得到的网页内容，并根据分析的数据生成 Item

Downloader Middleware 的用法

Downloader Middleware
处于 Scrapy 的 Request, Response 之间的模块

利用此中间件，主要可中断2个过程：

{Scheduler  Request→  DownloaderSpider  Response←  Downloader \begin{cases} Scheduler \;\underrightarrow{\text{Request}} \;Downloader\\ Spider\; \underleftarrow{\text{Response}}\;Downloader \end{cases} {SchedulerRequest​DownloaderSpiderResponse​Downloader​

所以 Downloader Middleware 的功能十分强大，比如：

  1. 修改 User-Agent
  2. 处理重定向
  3. 设置代理
  4. 失败重试
  5. 设置 Cookies

可以有多个中间件依序排列在 Spider, Downloader 中间，依序对 Request, Response 进行修改。

每个中间件有3个方法：(需要至少实现一个方法)

  1. process_request(request, spider)
  2. process_response(request,response,spider)
  3. process_exception(request,exception,spider)
    1. 当 Downloader 或 process_request() 方法抛出异常时被调用

Spider Middleware 的用法

和 Downloader Middleware 相似

有3个作用：

{Spider  Request→  SchedulerSpider  Response←  DownloaderItem  Pipeline  Item←  Spider \begin{cases} Spider\;\underrightarrow{Request}\;Scheduler\\ Spider\;\underleftarrow{Response}\;Downloader\\ Item\;Pipeline\;\underleftarrow{Item}\;Spider \end{cases} ⎩⎪⎨⎪⎧​SpiderRequest​SchedulerSpiderResponse​DownloaderItemPipelineItem​Spider​

Spider Middleware 的使用频率不如 Downloader Middleware 高，在必要的情况下它可以用来方便数据的处理。

Item Pipeline 的用法

发生时刻：

Item  Pipeline  Item←  Spider Item\;Pipeline\;\underleftarrow{Item}\;Spider ItemPipelineItem​Spider

主要功能:

  1. 清理 HTML 数据
  2. 验证爬取数据，检查爬取字段
  3. 查重、丢弃重复内容
  4. 将爬取结果保存到数据库

核心方法

  1. process_item(item, spider): 必须实现的方法，被默认调用对 Item 进行处理
  2. open_spider(self,spider): Spider 开启时自动调用。可做初始化工作，如开启数据库连接等
  3. close_spider(spider): Spider 关闭时自动调用。可做收尾工作，如关闭数据库连接等
  4. from_crawler(cls, crawler): 是一个类方法，用 @classmethod 标识，是一种依赖注入的方式
    1. 通过 crawler 可以拿到 Scrapy 的所有核心组件
    2. cls 是 Class，最后返回一个 Class 实例
