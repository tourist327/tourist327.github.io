前言

这一章主要介绍了一些基本概念，包括但不限于：

  • 最早的程序员
  • 编译程序的定义：简而言之就是一种语言翻译程序
  • （源、目标、宿主）（语言、程序、机）的区分
  • 编译的基本流程

源、目标、宿主概念的区分

语言：

源 目标 宿主
编写源程序 目标程序的描述语言 编译程序的实现语言

程序&机：

源程序 目标程序 宿主机（目标机）
源语言写的程序 源程序编译后的程序 编译程序的运行环境

基本流程

词法分析：源程序->属性字流

语法分析的结果的通常表示：

  • 分析树 parse tree
  • 语法树 syntax tree

语义分析：编译程序完成的最实质性的翻译工作

中间代码生成：

  • 特点
    • 易于产生
    • 易翻译为目标程序
  • 常见的有
    • 四元式
    • 三元式
    • 间接三元式
    • 逆波兰等

代码优化：改进目标代码质量

目标代码生成

编译程序的表示

  • 函数表示： T=C(S)
    • T: 目标程序
    • C: 编译程序
    • S: 源程序
  • T 型图
  • 符号表示

编译器实现的途径

  1. 手工构造
  2. 自动构造工具
  3. 自展方式
  4. 移植方式

几个概念

表格管理：

  • 编译程序的公共辅助程序
  • 管理源程序中各种量、登记相应表格

出错处理：

  • 编译程序的公共辅助程序
  • 检查、定位、定性、报告：源程序中的各种错误
  • 尽可能将错误限制在尽可能小的范围内，保障编译继续

遍（Pass)或趟
